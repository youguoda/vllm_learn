## 📚 知识详解（快速上手 · 2026-06-12 补充）

### PP（流水线并行）：按层切，跨节点的选择
TP 切矩阵内部（通信密集、节点内），PP 把**整层**分段放到不同卡/节点（通信只在段边界传激活，量小）→ 适合跨机扩展。

```
卡0: 层1-8 → 传激活 → 卡1: 层9-16 → ... → 卡3: 层25-32
问题：同一时刻只有一段在干活 = 流水线气泡
解法：微批次(micro-batch)填充 —— 请求切成小批连续注入，
      卡0 算完批1立刻算批2，各段错峰忙碌
```

### 推理 PP 与训练 PP 的不同（容易混）
推理没有反向传播，气泡问题轻得多；continuous batching 天然提供了源源不断的"微批"。难点变成：**段间负载均衡**（每段计算量要接近，否则慢段拖全队）和首 token 延迟（要穿过全部段）。

### 两框架对照
| | vLLM | SGLang |
|--|------|--------|
| 参数 | `--pipeline-parallel-size` | `--pp-size` |
| 典型组合 | 节点内 TP × 跨节点 PP | 同 |
| 延迟特征 | TTFT 增加（穿越各段） | 同 |

### 本周综合对比表（TP/PP/EP/DP 收官）
| 策略 | 切什么 | 通信 | 解决什么 | 代价 |
|------|--------|------|---------|------|
| TP | 矩阵内部 | all-reduce 每层2次 | 权重放不下 | 通信税重，限节点内 |
| PP | 层段 | 段间传激活 | 跨节点扩容 | 气泡+TTFT 增 |
| EP | 专家 | all-to-all | MoE 专家放不下 | 通信不均衡 |
| DP | 不切(副本) | 无 | 纯扩吞吐 | 显存×N |
