## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 社区争论的 5 个经典议题（按这个清单去搜）
1. **"APC 追平 RadixAttention 了吗"**：vLLM 块哈希持续优化；分歧点在低复用率与非对齐前缀场景。搜：vLLM GitHub issues `prefix caching performance`
2. **缓存开销问题**：0% 复用负载下，哈希计算/树维护是纯开销——两边各有 benchmark 互指对方慢，注意看测试负载是否公平
3. **调度公平性**：SGLang cache-aware 调度可能饿死低命中请求？（长尾 P99 视角）
4. **v1 引擎重写**：vLLM v1 对调度器/内存管理的重构大幅缩小差距，**旧文章的结论很多已过期，看文先看日期和版本号**
5. **多租户隔离**：共享缓存是否泄漏租户间信息（前缀命中时间侧信道）——安全视角的少数派讨论

### 信息源质量排序
官方 blog/论文 > 框架 GitHub issue 里的 benchmark 帖 > 知乎/Medium 深度长文 > 营销号对比文（直接跳过）

### 整理时的去伪存真三问
1. 测试用的什么版本？（差一个大版本结论可能反转）
2. 负载是否偏向某框架的甜区？（全是共享前缀 = 给 SGLang 喂饭）
3. 有没有可复现的脚本？（没有脚本的数字当传闻处理）
