### "vLLM 初体验" PPT 素材骨架（直接可用）

**叙事线：痛点 → 原理 → 实测 → 适用**

1. **痛点页**：一张图说明 decode 是访存瓶颈 + KV Cache 预分配浪费 60-80%（画两条显存条对比）
2. **原理页**：Block Table 示意图（参考 4/25 的流程图）+ 一句话类比"OS 虚拟内存"
3. **三大支柱页**：PagedAttention（装得多）/ Continuous Batching（不空转）/ Prefix Caching（不重算）
4. **实测页**：自己跑的数字最有说服力——`vllm bench serve` 在 1/8/32 并发下的吞吐和 TTFT 表格
5. **生态页**：OpenAI 兼容 API、HF 模型即插即用、量化支持 → "为什么它成了事实标准"

### 讲给听众的3 个记忆点（Takeaway 句式）

- "vLLM 的本质是**用内存效率换吞吐量**"
- "并发上不去时，先看 KV Cache 池，再看算力"
- "共享前缀的业务（客服、Agent）开 Prefix Caching 白拿 TTFT 收益"
