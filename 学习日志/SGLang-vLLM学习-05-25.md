## 📚 知识详解（快速上手 · 2026-06-12 补充）

### KV Cache 量化：把"并发天花板"抬高一倍
权重量化省的是固定成本，**KV 量化省的是随上下文增长的变动成本**——长上下文/高并发场景它才是主角。

```bash
# vLLM
vllm serve $M --kv-cache-dtype fp8   # fp8_e4m3(精度优先)/fp8_e5m2(范围优先)
# SGLang
python -m sglang.launch_server --model-path $M --kv-cache-dtype fp8_e5m2
```
直接效果：KV 显存减半 → 同卡并发/可用上下文翻倍；代价：attention kernel 需支持量化 KV 读取（反量化融合在 kernel 里），精度损失通常可忽略，**长链推理任务建议实测 PPL/任务分**。

### 长上下文场景的账（32K 上下文，7B GQA 模型）
| | FP16 KV | FP8 KV |
|--|---------|--------|
| 单请求 32K KV | ~1.8 GB | ~0.9 GB |
| 24GB 卡可并发 | ~8 路 | ~17 路 |
