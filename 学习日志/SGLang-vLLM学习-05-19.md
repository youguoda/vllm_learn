## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 投机解码：用小赌大，无损加速
decode 慢的根源：每个 token 都要把**整个模型权重**从显存读一遍（访存瓶颈）。
**Draft-Verify 范式**：便宜的"草稿员"快速猜 k 个 token → 大模型**一次前向**并行验证 k 个 → 接受匹配的前缀，第一个不匹配处由大模型给出正确 token。

为什么无损：验证用拒绝采样，**数学上保证输出分布与大模型独跑完全一致**——加速不掉精度，这是它和量化的本质区别。

收益公式直觉：`加速比 ≈ 平均接受长度 / (1 + 草稿成本占比)`，接受率是命门。

### EAGLE（当前主流，重点读）
- 不用独立小模型，在大模型**最后一层特征**上接一个轻量自回归头做草稿（特征级比 token 级好猜）
- EAGLE-2/3 引入**动态草稿树**：一次猜一棵树（多条候选路径），用 tree attention 一次验证整棵树 → 接受长度更长
- 典型加速 2-4×（batch 小、温度低时最佳）

### Medusa（理解思想即可）
在大模型头上并联多个 **MLP 头**，第 i 个头直接猜第 i 步 token（非自回归、互相独立）→ 结构最简单，但接受率低于 EAGLE，已基本被取代。

### 高并发下的反直觉结论（实验周会验证）
batch 大时 GPU 算力本就吃满，验证的"并行红利"消失，投机解码收益骤降甚至为负 → **它是低并发/延迟敏感场景的武器，不是吞吐武器**。
