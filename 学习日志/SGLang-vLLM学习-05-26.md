## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 指标的精确定义（benchmark 的地基，差之毫厘谬以千里）
| 指标 | 定义 | 易错点 |
|------|------|--------|
| TTFT | 发出请求→收到第1个token | 必须含排队时间（客户端测，别用服务端日志） |
| TPOT | (总时长-TTFT)/(输出tokens-1) | 输出只有1个token时无定义 |
| ITL | 相邻 token 间隔的分布 | 看 P99 抓卡顿，均值会骗人 |
| Throughput | 系统总输出 tok/s | 区分 output tok/s 与 total tok/s |
| **Goodput** | 满足 SLO（如 TTFT<1s）的请求吞吐 | 比裸吞吐更接近业务真相 |

### 负载模式：request-rate 比并发数更科学
固定并发=闭环（上一条回来才发下一条，系统慢→发得慢→互相掩护）；**固定到达率（泊松过程）=开环**，更像真实流量。主流工具都支持 `--request-rate`，扫描 rate 直到系统饱和。

### 工具盘点
| 工具 | 优点 | 用途 |
|------|------|------|
| `vllm bench serve` | 官方、指标全 | 主力 |
| `sglang.bench_serving` | 同上，支持多后端 | 主力（建议统一用一个工具测两家！） |
| genai-perf (NVIDIA) | 中立第三方 | 交叉验证 |
| 自写 asyncio 脚本 | 灵活构造业务负载 | 多轮/前缀场景 |

**关键决策：统一用同一个压测工具打两个框架的 OpenAI 端点，排除工具偏差。**
