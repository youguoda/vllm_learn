## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 社区数据去哪找（按可信度排序）
1. 两家官方 blog 的 benchmark 文章（注意各自立场）
2. GitHub 仓库的 `benchmarks/` 目录和 performance 标签 issue
3. 第三方独立评测：NVIDIA genai-perf 文章、云厂商选型报告、BentoML/Anyscale 等工程博客
4. 论文复现仓库（SGLang 论文附录、vLLM SOSP 论文数据）

检索词模板：`vllm sglang benchmark 2026`、`site:github.com vllm performance regression`

### 差异原因排查清单（按命中率排序）
1. **版本差**：差一个 minor 版本结论可能反转（v1 引擎前后的 vLLM 尤其）
2. **负载偏置**：共享前缀比例、输入输出长度分布
3. **硬件差**：H100 与消费卡/国产卡的访存算力比完全不同 → kernel 优劣关系会变
4. **参数没对齐**：缓存开关、量化、max-len
5. **测量口径**：闭环 vs 开环、是否含排队

### 写进报告的方式
附录一节「与社区数据的交叉验证」：一致的增强可信度，不一致的给出原因假设——**敢于展示不一致并解释，比只贴一致数据可信十倍**。国产卡环境的差异尤其值得写，这是你独有的数据。
