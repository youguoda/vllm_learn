## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 量化全景图（先建坐标系再记细节）
```
量化什么：权重 W / 激活 A / KV Cache（明天）
量化到几位：FP8 ≈ 无损 | INT8 | INT4（有损，需校准算法）
怎么标定：AWQ / GPTQ / RTN（算法只在"怎么选 scale/保护谁"上不同）
```
命名速记：**W4A16** = 权重4bit、激活16bit（即 AWQ/GPTQ 类 weight-only）；**W8A8** = 权重激活都8bit（FP8/INT8）。

### 三种主流方案的一句话原理
- **FP8（E4M3）**：直接换浮点格式+per-tensor/per-channel scale，精度损失通常 <0.1%，H100/Ada 有硬件加速 → **新卡首选**
- **GPTQ**：逐层最小化量化误差的二阶方法（用 Hessian 信息逐列量化并补偿）→ 4bit 经典
- **AWQ**：观察到 ~1% 的"显著权重"决定质量，按**激活分布**找出它们并用缩放保护 → 4bit 更稳、免反向

### 收益的真相（分享必讲）
weight-only 4bit 的加速来自**访存减半/减四**（decode 是访存瓶颈），不是算力——所以小 batch 加速明显（Marlin kernel），大 batch 转为算力瓶颈时收益缩小；FP8 W8A8 则是真算力加速（Tensor Core 原生）。

### 两框架支持速查
| 方案 | vLLM | SGLang | 用法 |
|------|------|--------|------|
| FP8 | ✅ | ✅ | `--quantization fp8` / 加载 FP8 checkpoint |
| AWQ/GPTQ | ✅(Marlin) | ✅ | 直接加载 HF 上的 -AWQ/-GPTQ 模型 |
| 校准 | 用现成量化模型最省事 | 同 | llm-compressor 可自制 |
