## 步骤 1：确认硬件限制，制定实验范围（20 min）

RTX 3060 是 Ampere 架构（sm_86），先确认支持什么：

```python
# check_hw.py
import torch
if __name__ == "__main__":
    print(f"GPU: {torch.cuda.get_device_name(0)}")
    print(f"Compute capability: {torch.cuda.get_device_capability(0)}")
    # Ampere = 8.x，Hopper = 9.x
    cap = torch.cuda.get_device_capability(0)
    print(f"\n量化支持情况：")
    print(f"  FP8 计算（W8A8）: {'✓' if cap >= (9,0) else '✗ 需要 Hopper (sm_90+)'}")
    print(f"  INT8 计算: {'✓' if cap >= (8,0) else '✗'}")
    print(f"  INT4 (AWQ/GPTQ 权重): ✓ (dequant to fp16 计算)")
    print(f"  FP8 存储（KV cache fp8_e5m2）: ✓ (Ampere 支持存储格式，计算用 fp16)")
```

**结论**（写进笔记）：
- ❌ FP8 W8A8 推理：需要 Hopper（H100），3060 不支持
- ✓ AWQ INT4：权重 4bit 存储，dequant 后 fp16 计算，3060 支持
- ✓ GPTQ INT4：同上
- ✓ FP8 KV cache（e5m2 格式）：只是存储格式，计算仍 fp16，3060 支持

```python
from awq import AutoAWQForCausalLM
from transformers import AutoTokenizer

if __name__ == "__main__":
    model_path = "Qwen/Qwen2.5-1.5B"   # 已缓存的 fp16 版
    quant_path = "/tmp/qwen1.5b_awq"
    quant_config = {"zero_point": True, "q_group_size": 128,
                    "w_bit": 4, "version": "GEMM"}
    model = AutoAWQForCausalLM.from_pretrained(model_path, device_map="auto")
    tokenizer = AutoTokenizer.from_pretrained(model_path)
    model.quantize(tokenizer, quant_config=quant_config)
    model.save_quantized(quant_path)
    tokenizer.save_pretrained(quant_path)
    print(f"已保存到 {quant_path}")
```

量化完后跑对比实验：

```python
# compare_quant.py
import time, torch
from vllm import LLM, SamplingParams

PROMPT = "请详细解释工业传感器中压电式加速度计的工作原理，包括物理原理、结构设计、信号处理流程和典型应用场景。"
PARAMS = SamplingParams(temperature=0.0, max_tokens=300)

def bench(model_path, label, n=3):
    print(f"\n加载 {label}...")
    mem_before = torch.cuda.memory_allocated() / 1e9
    llm = LLM(model=model_path, max_model_len=4096, gpu_memory_utilization=0.85)
    mem_after = torch.cuda.memory_allocated() / 1e9
    print(f"  模型显存: {mem_after - mem_before:.2f} GB")

    # 预热
    llm.generate([PROMPT], PARAMS)
    # 正式计时
    times = []
    for _ in range(n):
        t0 = time.time()
        out = llm.generate([PROMPT], PARAMS)
        times.append(time.time() - t0)
    avg = sum(times) / n
    ntok = len(out[0].outputs[0].token_ids)
    print(f"  平均耗时: {avg:.2f}s，{ntok} token，{ntok/avg:.1f} tok/s")
    # 质量对比：打印前50字
    print(f"  输出预览: {out[0].outputs[0].text[:80]}...")
    del llm; torch.cuda.empty_cache()
    return mem_after - mem_before, ntok/avg

if __name__ == "__main__":
    results = {}
    results["fp16"] = bench("Qwen/Qwen2.5-1.5B", "FP16")
    results["awq"]  = bench("/tmp/qwen1.5b_awq", "AWQ INT4")

    print("\n=== 对比汇总 ===")
    print(f"{'版本':<10} {'模型显存(GB)':<15} {'tok/s':<10}")
    for k, (mem, tps) in results.items():
        print(f"{k:<10} {mem:<15.2f} {tps:<10.1f}")
    fp16_mem, fp16_tps = results["fp16"]
    awq_mem,  awq_tps  = results["awq"]
    print(f"\n显存节省: {(1-awq_mem/fp16_mem)*100:.0f}%")
    print(f"速度变化: {awq_tps/fp16_tps:.2f}x (>1 更快, <1 更慢)")
```

## 步骤 3：AWQ vs GPTQ 原理笔记（30 min）

不需要都实测，读原理写进笔记：

| | AWQ | GPTQ |
|---|---|---|
| 核心思路 | 找"重要通道"（激活值大的），对这些通道权重少量化 | 逐层最小化量化误差（二阶近似，Hessian） |
| 量化过程 | 逐层分析激活分布，给通道加缩放因子 | 逐层迭代更新权重，补偿量化误差 |
| 量化速度 | 快（几分钟） | 慢（十几分钟到数小时） |
| 精度 | 略低（保护重要通道） | 略高（逐层误差最小化） |
| 推理 | dequant 到 fp16 再计算（无需特殊硬件） | 同 |

## 步骤 4：理解"INT4 省一半显存但速度未必快一半"（10 min）

写进笔记三点：
1. **decode 是访存瓶颈**：每步 decode 要把权重从显存搬到计算单元，搬的字节少了→快；但 dequant（INT4→fp16）有额外计算开销，部分抵消
2. **prefill 是计算瓶颈**：prefill 时 batch 大，算力利用率高，减少权重 IO 帮助有限
3. **实际加速**：1.5B 小模型 decode 本身很快（<10ms/token），dequant 开销比例更大；大模型（70B+）decode 很慢（每步要搬很多权重），AWQ 加速比更明显
