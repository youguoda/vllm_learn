```markdown
## 5. 结论与选型建议

### 5.1 核心结论

**结论1（通用负载）**：无共享前缀时两框架吞吐/延迟差异在 ___% 以内，
均属正常实现差异，不构成选型依据。

**结论2（高复用负载）**：多轮对话等高前缀复用场景下，SGLang 凭借
RadixAttention 在第 2 轮起将 TTFT 降低 ___ms（___%)。前缀越长、复用率越高，
优势越显著（05-10 数据：复用率 100% 时 TTFT 差异 ___ms）。

**结论3（进阶特性）**：结构化输出两框架均达 100% 合法率、热身后速度损耗 <5%；
量化（AWQ INT4）节省 ___% 显存；FP8 KV cache 使可缓存 token 翻倍，
在长上下文场景下意义重大。

### 5.2 选型建议

| 场景 | 推荐 | 原因 |
|---|---|---|
| 通用批量推理，无共享前缀 | vLLM | 生态更成熟，两者性能持平 |
| 多轮对话 / RAG / Agent | SGLang | RadixAttention TTFT 低 ___% |
| 结构化输出（固定 schema）| 两者均可 | 性能无本质差异 |
| 长上下文（>4K），显存受限 | SGLang + FP8 KV | Radix + KV 量化双重收益 |
| 需要 MoE + EP 并行 | SGLang | 内置 EP/DP Attention |

### 5.3 外推局限

本实验在 RTX 3060 12GB + 1.5B 模型下进行，以下结论**不可直接外推**：
- 并行策略（TP/PP/EP）相关结论：单卡无法实测
- 绝对延迟数字：大模型（7B+）延迟高一个数量级
- FP8 计算加速：需 H100，本实验环境不支持

以下结论**可外推**（与规模无关的定性结论）：
- "高复用负载 SGLang 更优" 的趋势
- TTFT 与输入长度线性的关系
- KV 量化使可缓存量翻倍的数学结论
```
