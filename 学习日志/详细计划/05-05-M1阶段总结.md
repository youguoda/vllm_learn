**第一段：vLLM 三大支柱（Week 1 数据）**

| 支柱 | 机制一句话 | 我的证据 |
|---|---|---|
| PagedAttention | KV 分 16-token 块 + Block Table 间接寻址，浪费<4% | blocks 核账表（04-25/28） |
| Continuous Batching | token 级调度，请求随到随插 | 并发 1→20 RPS 提升 X 倍（04-27） |
| Prefix Caching | 整块哈希复用前缀 KV | TTFT 降 X%（04-26） |

**第二段：SGLang 两大特性（Week 2 数据）**

| 特性 | 机制一句话 | 我的证据 |
|---|---|---|
| RadixAttention | radix 树索引 KV，任意长度前缀命中 | 多轮对话轮 2 起 TTFT 降 X%（05-02），半块前缀对照表 |
| Structured Output | FSM 在 logits 层 mask 非法 token | 合法率 X%→100%，速度损耗 ~0（05-03） |

**第三段：初步对比结论（05-04 数据）**
- 无共享前缀的通用负载：两者吞吐/延迟接近（贴 compare_v1.png）
- 已观察到的真实差异：缓存粒度（半块前缀实验）、结构化输出易用性
- 待验证假设：前缀复用率越高 SGLang 优势越大 → Week 3 的 05-10 实验
