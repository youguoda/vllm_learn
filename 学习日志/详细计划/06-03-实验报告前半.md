```markdown
## 1. 背景与目标

### 1.1 问题来源
LLM 推理分两阶段：Prefill（并行处理输入，算力密集）和 Decode（逐 token 生成，
访存密集）。Decode 阶段每步只生成 1 个 token，GPU 的大量并行计算单元处于等待状
态——实测 RTX 3060 单请求 decode 时 GPU 利用率仅 ___\%（见 05-19 实验）。

要提升利用率，需要加大 batch size。但 batch 的瓶颈是 KV Cache 显存：传统方案为
每个请求预留最大长度的连续显存，浪费率达 60-80\%，导致同一张卡实际并发数极低。

vLLM 和 SGLang 分别用不同的方式突破这一瓶颈，本报告通过受控实验量化两者的差异。

### 1.2 研究问题
1. 无共享前缀的通用负载下，两框架吞吐/延迟是否有显著差异？
2. 高共享前缀的多轮对话负载下，差异是否如理论预期显著拉大？
3. 进阶特性（结构化输出、量化）对性能的影响各自如何？

### 1.3 局限声明
- 硬件：RTX 3060 12GB（Ampere），无法测试 FP8 计算、多卡并行
- 模型：Qwen2.5-1.5B，小模型某些结论不可直接外推到 70B+
- 版本：vLLM v___，SGLang v___，框架快速迭代，结论有时效性
```

### 2.2 测试场景

| 场景 | 目的 | 关键变量 |
|---|---|---|
| A. 固定输出 | 测吞吐上限和 TTFT-并发关系 | 并发数 {1,4,8,16,32} |
| B. 延迟 | 测 TTFT 与输入长度的关系 | 输入长度 {64,256,512,1024,2048} |
| C. 多轮对话 | 测缓存机制的实际收益 | 轮次 1-5，并发 {1,8,16} |
| D. 流式 ITL | 测 token 间延迟分布 | 并发 {1,4,8} |

### 2.3 评测指标

- **TTFT**（First Token Latency）：用户等待首个 token 的时间，影响感知响应速度
- **ITL**（Inter-Token Latency）：相邻 token 间隔，决定流式输出的"打字流畅度"
- **Throughput**（吞吐量）：系统每秒产生的总 token 数，决定服务成本
- **E2E**（端到端延迟）：完整响应时间

### 2.4 实验控制
- 每组 40 请求，3 次重复取中位数
- 每轮切换框架前重启 server，清除残留进程
- 预热 5 次请求后开始计时

## 步骤 4：写"架构对比"（45 min）

```markdown
## 3. 架构对比

### 3.1 KV Cache 管理机制（核心差异）

**vLLM PagedAttention**：
（配图 assets/pagedattention_arch.png）
将 KV cache 按 16-token 块管理，通过 Block Table 实现逻辑连续、物理离散的映射。
APC（Automatic Prefix Caching）为每个满块计算链式哈希，新请求命中整块哈希时跳过
该部分的 prefill 计算，命中粒度为 16 token 整块。

**SGLang RadixAttention**：
（配图 assets/radixattention_arch.png）
用 Radix 树索引 KV 缓存，树的边代表一段 token 序列，任意长度的公共前缀均可命中。
match_prefix() 从根节点逐边匹配，命中深度不受块边界限制；insert() 在必要时 split
节点，保证树的紧凑性；evict() 按 LRU 从叶子开始淘汰，lock_ref > 0 的节点不可淘汰。

**关键差异（配图 assets/kvcache_diff.png）**：

| 维度 | vLLM | SGLang |
|---|---|---|
| 命中粒度 | 16 token（整块） | 任意 token |
| 索引结构 | 哈希表 | Radix 树 |
| split 操作 | 无 | 有（边可劈开） |
| 设计哲学 | OS 虚拟内存（管好显存） | 缓存系统（最大化复用） |

### 3.2 调度器差异
（配图 assets/scheduler_compare.png）
vLLM：FCFS + token budget + preemption（recompute/swap）
SGLang：缓存感知调度，优先调度能命中 radix 树的请求，减少 prefill 总量

### 3.3 进阶特性支持对比

| 特性 | vLLM | SGLang |
|---|---|---|
| 结构化输出 | xgrammar/outlines | Compressed FSM |
| 投机解码 | ngram/EAGLE/draft model | EAGLE |
| 量化 | AWQ/GPTQ/FP8 KV | 同 |
| 多机并行 | TP+PP | TP+PP+EP+DP Attention |
```
