# 05-20 详细计划：投机解码框架实现对比

## 步骤 1：读 vLLM 投机解码实现（60 min）

```bash
VLLM=$(python3 -c "import vllm; print(vllm.__file__.replace('/__init__.py',''))")
# 找投机解码入口
grep -rn "speculative\|spec_decode\|ngram" $VLLM --include="*.py" -l | head -10
# v1 引擎的入口通常在：
ls $VLLM/v1/spec_decode/ 2>/dev/null || ls $VLLM/spec_decode/ 2>/dev/null
```

重点读三件事：

**① 支持的 draft 方案**
```bash
grep -rn "method.*ngram\|NgramSampler\|draft_model" $VLLM --include="*.py" | head -10
```
vLLM 支持：ngram（无额外模型，从 prompt 里找重复片段）、draft_model（独立小模型）、EAGLE（需要对应 draft head 权重）

**② 接受率统计在哪**
```bash
grep -rn "acceptance_rate\|accepted_tokens\|num_accepted" $VLLM --include="*.py" | head -10
```
找到接受率的计算逻辑，理解：接受率 α ≈ 0.8 时，平均每次验证接受 K×α 个 token，理论加速比 = K×α / (1 + K×verify_overhead_ratio)

**③ speculative config 格式**
```bash
grep -rn "SpeculativeConfig\|speculative_config" $VLLM --include="*.py" | head -5
```
记录 `--speculative-config` 的 JSON 字段，明天配置实验要用。

## 步骤 2：读 SGLang 投机解码实现（40 min）

```bash
source ~/venv-sglang/bin/activate
SGLANG=$(python3 -c "import sglang; print(sglang.__file__.replace('/__init__.py',''))")
grep -rn "speculative\|eagle\|draft" $SGLANG --include="*.py" -l | head -10
```

对比两框架的集成方式：

| 对比点 | vLLM | SGLang |
|---|---|---|
| ngram 支持 | `--speculative-config '{"method":"ngram",...}'` | `--speculative-algorithm NEXTN` 或类似 |
| EAGLE 支持 | 需要 EAGLE draft head 权重 | `--speculative-algorithm EAGLE` |
| 接受率暴露 | `/metrics` 里的指标名 | 日志/指标 |
| 与调度器集成 | 调度器感知 draft token 预算 | 类似 |

主要差异：两框架的投机解码都是"可选插件"而不是默认开启，集成点基本对称。

## 步骤 3：准备明天的实验配置（30 min）

明天（05-21）要跑 ngram 投机实验，今天准备两类 prompt：

```python
# prepare_spec_prompts.py — 生成两类测试 prompt
if __name__ == "__main__":
    # 类型1：高重复性（代码、模板、列表）— ngram 命中率高
    high_rep = [
        "请用 Python 实现冒泡排序，要求：\n1. 函数名为 bubble_sort\n2. 输入为列表\n3. 返回排序后的列表\n4. 包含详细注释",
        "请列出工业传感器的 10 种类型，每种包含：名称、工作原理、典型应用、优缺点",
        "请用以下格式描述 5 种轴承故障：故障名称：\n产生原因：\n振动特征：\n检测方法：\n维修建议：",
    ]
    # 类型2：自由创作（故事、分析）— ngram 命中率低
    low_rep = [
        "请写一篇关于未来城市交通的科幻短篇故事，要求情节新颖，人物丰满",
        "分析人工智能对工业制造业的深远影响，从技术、经济、社会三个维度展开",
        "描述一次复杂工业事故的调查过程，包括现场勘查、数据分析、根本原因分析",
    ]
    print("高重复 prompts（ngram 受益大）：", len(high_rep))
    print("低重复 prompts（ngram 受益小）：", len(low_rep))
    # 保存
    import json
    with open("spec_prompts.json", "w") as f:
        json.dump({"high_rep": high_rep, "low_rep": low_rep}, f, ensure_ascii=False, indent=2)
```

自测：ngram 投机的 draft 来源是哪里？不是小模型——是从**当前请求的 prompt 里**找与当前生成位置相似的 n-gram 片段，把后续 token 当作 draft。代码类（有大量重复语法结构）命中率高，自由创作（每句话都新）命中率极低。
