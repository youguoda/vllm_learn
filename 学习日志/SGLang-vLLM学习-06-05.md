## 📚 知识详解（快速上手 · 2026-06-12 补充）

### 按场景归类的分析框架（M3 主线）
把所有数据折叠成一张"场景胜负表"，每格只放三样东西：**赢家、差距幅度、置信度**。

| 场景 | 指标主导 | 赢家 | 差距 | 置信度 |
|------|---------|------|------|--------|
| 单请求低延迟 | TTFT+ITL | | | 高/中/低 |
| 高并发吞吐 | 拐点吞吐 | | | |
| 流式体验 | ITL P99 | | | |
| 多轮对话 | 逐轮TTFT | | | |
| 结构化输出 | 吞吐损失+可解析率 | | | |

置信度判据：重复跑过≥2次且方向一致=高；单次=中；数据有未解释异常=低。**低置信度的格子不进 PPT 主张，只进附录。**

### 对比分析的三个深挖方向（数据不只是排名）
1. **交叉点**：有没有"低负载 A 赢、高负载 B 赢"的反转？反转点本身就是选型阈值
2. **方差**：均值相近时看 P99 谁更稳——生产更怕抖动
3. **资源换算**：性能差 10% 但显存省 30%，结论可能反转（成本视角，明天展开）

### 产出物
一份《结论速记》md：每条结论一行，按 6/2 的四要素句式——这是后续可视化、报告、PPT 的**单一事实来源**，三处引用都从这里复制，防止数字漂移。
