# SGLang 与 vLLM：大模型推理框架对比与实践

## 项目概述

**分享时长：** 40-50 分钟
**目标听众：** 公司内部技术团队（AE/研发混合）
**分享时间：** 2026年7月（具体日期待定）

## 目标

- [ ] 建立对两个框架的深度技术认知（不仅是会用，要能讲清楚 why）

## 笔记

### 核心对比要点速查

| 维度 | SGLang | vLLM |
|------|--------|------|
| **核心创新** | RadixAttention（跨请求KV前缀复用） | PagedAttention（虚拟内存式KV管理） |
| **吞吐量** | 领先约 29%（Llama 3.1 8B, H100） | 单轮唯一prompt场景可反超 |
| **多轮/RAG** | 最高 6.4× 提升 | 无跨请求前缀复用 |
| **DeepSeek支持** | 官方推荐引擎 | 跟进中 |
| **硬件生态** | 主要NVIDIA，AMD增长中 | 最广：NVIDIA/AMD/TPU/Gaudi/CPU |
| **结构化输出** | Compressed FSM（~3× faster JSON） | 标准 constrained decoding |
| **GitHub Stars** | ~25K | ~75K |
| **贡献者** | ~600 | ~2,400 |
| **生产用户** | xAI、Azure、Cursor、LinkedIn | 几乎所有云服务商默认 |

