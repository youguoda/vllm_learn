# 交互可视化

> 把项目里最抽象的 6 个概念做成可以**拖、点、看动画**的交互页面,辅助理解。

## 怎么打开

**双击 `index.html`** 即可(纯 HTML+JS,无任何依赖,完全离线)。
或在 WSL 里:`explorer.exe index.html`(用 Windows 浏览器打开)。

## 6 个交互模块

| 模块 | 玩法 | 类比 | 帮你理解 |
|---|---|---|---|
| ① 命中粒度:块 vs token | 拖"截断点",看谁救回更多 token | 整箱搬货 vs 散装抓取 | vLLM 82.1% vs SGLang 94.9% 的根源 |
| ② Radix 树演化 | 点击加请求,看树 split 分叉 | 地铁线路图 | 任意位置 split,无块对齐约束 |
| ③ 内存浪费 | 拖并发数,对比传统 vs 分页 | 霸占整排车位 vs 智能车位 | 浪费 98.8%→21.9%,多服务 65 倍 |
| ④ 链式哈希 | 点一个块改它,看下游全失效 | 多米诺骨牌 | 差一 token 缓存全 miss |
| ⑤ Continuous Batching | 切 static/continuous 看动画 | 包车 vs 拼车 | 座位填满 = 吞吐高 |
| ⑥ 复用率→吞吐 | 拖复用率,看两线交叉 | 食堂预制菜比例 | 你的 2.1× 是怎么来的 |

## 数据来源

所有数字均为 **RTX 3060 12GB + Qwen2.5-1.5B** 本机实测(见 `../bench_results/` 和各实验笔记),不是编造的演示值。

## 推荐学习顺序

①(核心差异)→ ②(为什么能做到)→ ⑥(性能后果)是主线;③④⑤ 是补充理解 PagedAttention 和 Continuous Batching 的机制。
