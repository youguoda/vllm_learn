FSM 状态机（压缩前 vs 压缩后）
❌ 旧系统：普通 FSM（每步一个token）
✅ 新系统：Compressed FSM（确定性路径合并）
核心思路：
当 FSM 某段路径上每个状态只有 唯一 的合法下一token（确定性），就把这段路径压缩成一步，一次 forward pass 直接输出整段文字。

用打字类比：
普通：每敲一个键就问"下一个键是什么？"
压缩：已经知道接下来要敲 {"summary": " 这11个字符，直接一次全敲完。
两种方案对比
旧系统：逐 token decode
生成 {"summary": " (11个字符) 需要：
FP1: { FP2: " FP3: sum FP4: mary FP5: " FP6: : FP7: _" = 7次 Forward Pass
新系统：Compressed FSM
整段固定字符串，FSM 识别为确定性路径：
FP1: {"summary": "（整段） + 模型自由生成内容 + FP2: "}（结尾）
= 2次 Forward Pass（节省5次）
