三种技术横向对比
技术 解决的问题 核心机制 效果 适用场景
RadixAttention KV Cache 每次请求后丢弃，相同前缀重复计算 Radix Tree 维护历史KV，prefix match → 直接复用 TTFT ↓，Throughput ↑，内存共享 多轮对话、RAG、Few-shot、相同system prompt
Compressed FSM 结构化输出的JSON固定字符，每个token都做一次forward pass FSM 中确定性单出边路径合并，一步输出整段 JSON decode 吞吐 ↑ 1.6× JSON mode、结构化输出、固定格式模板
API Speculative Exec 黑盒API模型的多次gen()调用重复发送上下文 第一次调用超额生成，后续调用本地验证复用 input token费用降低~3× GPT-4等API模型的多字段提取、信息抽取
三者的共同逻辑：
都是在"提前知道某些信息"的前提下，跳过不必要的计算。
RadixAttention → 知道"前缀相同"，跳过前缀的 prefill
Compressed FSM → 知道"下一个token确定"，跳过那次 forward pass
API Speculative → 知道"后续gen()会问同样的上下文"，提前多生成并本地复用
