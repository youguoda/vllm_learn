Chain Hash：每个 block 的哈希依赖前面所有 block
每个 block（16 token）的哈希值 = hash(前一个block的哈希, 本block的token序列)
→ 只要前缀中任意一个 token 变了，后面所有 block 的哈希都会变，保证唯一性
结论：请求②共4个block，其中Block 0和1与请求①完全相同（同样的系统提示），hash匹配 → 直接复用KV Cache，跳过这32个token的prefill计算。只需计算新的Block 2'和3'。

vLLM APC 的核心局限：必须对齐到 16-token block 边界
直觉理解：
vLLM APC 像是"只能按整张纸复印"——你有18页文档，只能复用前16页，最后2页还是要手抄。
SGLang RadixAttention 像是"可以逐页复印"——18页全复用，一页都不浪费。
